# 路线图

> 100 步，10 个阶段，1 个目标：AI-native 操作系统。

---

## 当前进度

我们正在 **Phase 1（起步阶段）**。

已完成: 开发环境搭建、核心文档、`run_shell()` 原型、第一个 AI → Linux 原型。

---

## Phase 1: 起步（1-10）

搭建环境，学习基础，写出第一个 AI→Linux 原型。

**产出:** ShuoZi Runtime v0.1

---

## Phase 2: Build in Public（11-20）

建立公开存在，培养持续输出习惯。

**产出:** 公开路线图、Dev Log、项目结构

---

## Phase 3: Tool Runtime（21-30）

构建结构化 tool calling 系统——ShuoZi OS 的核心抽象。

**产出:** Tool Runtime v0.2

---

## Phase 4: Linux 深入（31-40）

深入理解 Linux 内核机制（`/dev`, `/proc`, systemd, DBus, Docker），为后续自建内核积累知识。

**产出:** Linux Runtime Alpha

---

## Phase 5: 硬件层（41-50）

AI 直接与硬件交互：USB、串口、ADB、ESP32 烧录。

**产出:** Hardware Runtime v0.1

---

## Phase 6: 社区启动（51-60）

建立开发者社区，开放贡献，形成第一批核心用户。

---

## Phase 7: Context Engine（61-70）

AI 的"内存管理系统"——持久化、可压缩、可恢复的上下文。

**产出:** Context Engine Alpha

---

## Phase 8: Workflow Engine（71-80）

任务规划、执行、失败恢复——AI 的"进程调度器"。

**产出:** Workflow Runtime

---

## Phase 9: Multi-Agent（81-90）

多 agent 协作：研究 agent、编码 agent、硬件 agent。

**产出:** Multi-Agent Alpha

---

## Phase 10: 生态与融资（91-100）

开放平台，建立开发者生态，启动公司化运营。

**产出:** ShuoZi OS Beta

---

## 唯一 KPI

路线图里真正重要的不是某个功能、多少用户、融多少钱。

**真正的护城河 = 连续 10 年的系统积累。**

一个功能可以被模仿。10 年的内核代码、架构决策、tool 生态——无法被短期复制。

**所以唯一 KPI 是：不要停。**
