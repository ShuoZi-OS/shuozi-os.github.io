# ShuoZi OS — 双终端架构设计

> 版本: v1.0 | 日期: 2026-05-18 | 状态: Phase 1 原型

---

## 核心理念

ShuoZi OS 有两个终端。它们不是两个独立程序，而是一个系统的两种视图：

```
┌──────────────────────────────────────────────┐
│                  Session                      │
│  ┌────────────────────────────────────────┐  │
│  │         Tool Registry（共享）            │  │
│  │  shell.run  fs.read  fs.ls  fs.write   │  │
│  │  system.info  ...                       │  │
│  ├────────────────────────────────────────┤  │
│  │         State（共享）                    │  │
│  │  • 执行历史                              │  │
│  │  • 当前工作目录                          │  │
│  │  • 环境变量                              │  │
│  │  • 活跃的 tool call                      │  │
│  ├────────────────────────────────────────┤  │
│  │         Event Bus（共享）                │  │
│  │  • 一个终端执行 → 另一个终端看到         │  │
│  └────────────────────────────────────────┘  │
│                                               │
│   ┌──────────────┐       ┌──────────────┐     │
│   │  CLI 终端     │       │  AI 终端      │     │
│   │  (人类)       │       │  (Agent)      │     │
│   │              │       │              │     │
│   │ shuozi →     │       │ { "intent":  │     │
│   │ ls /         │       │   "list /" } │     │
│   │              │       │              │     │
│   │ ✓ ls /  2ms  │       │ {"ok":true,  │     │
│   │ bin  boot    │       │  "data":...} │     │
│   └──────────────┘       └──────────────┘     │
└──────────────────────────────────────────────┘
```

---

## 两个终端

### CLI 终端（人类）

- **外观**: 类 Windows 终端，命令行提示符，彩色输出
- **输入**: 自然语言或命令
- **输出**: 格式化表格、彩色状态图标、人类可读耗时
- **使用场景**: 开发者调试、手动操作、学习

### AI 终端（Agent）

- **外观**: 纯输入框，无装饰
- **输入**: JSON 结构化意图
- **输出**: JSON envelope（ok / data / error / meta）
- **使用场景**: AI agent 自动调用、程序化消费、pipe 串联

---

## 数据同步规则（不可违反）

1. **同一个 Session。** CLI 终端和 AI 终端共享同一个 Session 对象。不是一个读数据库另一个读缓存——是同一个内存对象。

2. **操作可见。** CLI 终端执行的任何命令，AI 终端通过 `history` 立即可查询。反之亦然。

3. **状态一致。** 工作目录、环境变量、活跃 tool——修改在任何一端立即反映到另一端。

4. **事件驱动。** 一个终端执行操作后，Session 发事件。另一个终端订阅事件，无需轮询。

5. **权限统一。** capability token 在 CLI 终端签发后，AI 终端能直接使用。不存在"CLI 终端特权高于 AI 终端"。

## 为什么不是"一个终端两种模式"

如果只做一个终端，通过 `--mode human|ai` 切换，表面上简单了，但丢失了关键能力：

- **同时操作。** 人可以在 CLI 终端调试，AI agent 同时在 AI 终端执行自动化任务。两不干扰。
- **审核追踪。** AI 终端做了什么，人通过 CLI 终端实时看到。这是安全模型的基础。
- **能力互补。** 人做 AI 不擅长的事（判断、创意），AI 做人慢的事（批量、重复）。两个终端各司其职。

这是 OS 级别的设计，不是 UI 级别的便利功能。
