# ShuoZi OS — 架构原则

> 这些是我们的设计宪章。写任何代码、做任何架构决策之前，先读这份文件。
> 如果你要破坏其中一条原则，你必须写 ADR（架构决策记录）说明理由。

---

## 七大核心原则

### 1. CLI 优先，永不做 GUI 优先

所有系统能力首先通过命令行、API、IPC、网络协议暴露。GUI 是这些接口的消费者，不是能力定义者。

**禁止:** 把功能逻辑写在 UI 层  
**允许:** 先有 `shuozi disk format --fs ext4`，后有图形化的磁盘格式化工具

### 2. 工具化架构

每一个系统能力封装为独立的、可组合的 tool。AI 通过结构化 tool call 操作系统，人类通过 CLI 操作系统。两者用同一个接口。

```
错误: "打开设置 → 点击网络 → 选择 WiFi → 输入密码"
正确: tool "network.wifi.connect" { ssid, password }
```

### 3. 能力抽象，非应用抽象

用户（人和 AI）想完成的是任务，不是操作软件。系统提供的是完成任务的能力。

```
"flash firmware to ESP32"  ← 能力
"open esptool, select COM3, click flash"  ← 软件操作（禁止）
```

### 4. 安全边界在架构里，不在注释里

权限控制、隔离、沙箱不是 Afterthought。每个能造成破坏的操作从设计第一天就带权限检查点。

实现方式: capability-based security。持有 capability token 才能执行危险操作。token 可以传递、但不可伪造。

### 5. 硬件是头等公民

ShuoZi OS 不只是跑在 QEMU 里的内核。它必须能直接与真实硬件交互：USB 设备、串口、Android 设备（通过 ADB）、单片机、开发板。

硬件能力也必须通过 tool 接口暴露。

### 6. 组合优于集成

每一个 tool 必须能独立使用。tool 之间通过统一协议通信，不存在"你只能用 A 搭配 B"的隐性耦合。

```
tool_a | tool_b | tool_c    ← 好的
只有 tool_a + tool_b 组合才能用 tool_c  ← 禁止（除非物理上必然耦合）
```

### 7. 不可逆决策用 ADR 记录

"为什么选这个调度算法""为什么用这个 IPC 模型""为什么不用 Linux 内核"——这类决策必须在 `docs/architecture/decisions/` 下留下记录。

## 系统设计原则（来自 Saltzer & Kaashoek）

### S1: 模块化

系统只有一个划分方式：高内聚低耦合。模块通过窄接口通信，模块内部实现是模块自己的秘密。

**指标:** 一个模块的改动不应迫使其他模块改动。如果经常发生，就是模块边界画错了。

### S2: 最小惊讶原则

接口行为必须符合调用者的合理预期。

```
get_time() -> 返回秒级精度             ← 预期
get_time() -> 返回纳秒但只有 30% 正确率  ← 反模式（或者别叫 get_time）
```

### S3: 显式优于隐式

副作用必须显式声明。`write_file(path, data)` 这个函数名不提示它可能创建文件——应该在文档里说清楚，或者函数签名本身暴露这个事实。

### S4: 原子化操作

操作要么完全成功，要么完全失败。不存在中间态。

```
move_file(from, to): 失败后 from 里文件还在  ← 原子
move_file(from, to): 失败后 from 空了 to 也没文件  ← 灾难
```

## 决策框架

当你面对一个架构选择时，按这个顺序思考：

1. **这是不可逆决策吗？** 是 → 写 ADR。否 → 可以快速决定但要在 commit message 里写理由。
2. **违反了哪条原则？** 诚实回答。违反原则不是绝对的禁止，但要有充分理由（写在 ADR 里）。
3. **3 年后回头看，这个决策会让我后悔吗？** 如果答案是"可能"，再想一想。
4. **有人比我更懂这个领域吗？** 有 → 先问。没有 → 决策 + ADR。

## 反模式清单

以下是永远不应该出现在 ShuoZi OS 代码库里的东西：

| 反模式 | 为什么禁止 |
|--------|-----------|
| 全局可变状态 | 并发 bug 的工厂 |
| "万能函数"（超过 100 行） | 难以测试、难以推理、难以复用 |
| 重复代码（拷贝粘贴） | 修一处忘一处 |
| 字符串打字错误处理 | 编译器不会帮你检查 |
| 注释说"这里有个 bug" | 不修就别写，写 issue |
| 预留未实现的接口 | YAGNI。需要的时候再加。 |
| 硬编码路径 | 配置化 |
| 同步阻塞 I/O（AI runtime 层） | AI agent 不能卡住等 I/O |
