# ShuoZi OS — 愿景

## 我们为什么存在

2026 年，AI 已经很聪明了。它能推理、能编程、能写文章。但它没有自己的身体。

每次 AI 想操作电脑，它必须经过一个尴尬的翻译层：把意图转成鼠标点击坐标、把系统状态转成截图像素。它在模拟人类操作 GUI，而不是直接操作系统。

这不是 AI 的问题。这是 OS 的问题。

**ShuoZi OS 要解决的就是这个。** 一个 OS，它的设计目标不是"让人类用鼠标操作"，而是"让 AI 通过结构化接口操作系统"。

---

## 10 年后的世界

2036 年。你打开一台运行 ShuoZi OS 的机器。

你说："帮我搭建一个监控我菜园土壤湿度的系统。传感器数据通过 ESP32 上传，当湿度低于阈值时自动浇水。"

AI 做的事情：

1. 通过 `tool "serial.flash"` 把程序烧进 ESP32
2. 通过 `tool "device.provision"` 注册设备、分配 capability token
3. 通过 `tool "agent.spawn"` 启动一个监控 agent，订阅传感器数据流
4. 通过 `tool "agent.spawn"` 启动一个控制 agent，高阈值触发时调 `tool "gpio.write"` 打开水泵
5. 通过 `tool "ui.dashboard"` 生成一个实时监控页面（这是 GUI 的唯一用途：展示）

全过程没有打开一个 IDE。没有配置编译工具链。没有写 systemd unit 文件。没有手工配网络。

**这就是 AI-native OS。**

---

## 我们不是什么

- 不是 Linux 发行版。我们最终不依赖 Linux 内核。
- 不是聊天机器人。AI 是 OS 的一等公民，不是 OS 上的一个应用。
- 不是"自动化工具"。我们的目标是 OS 本身的重新定义，不是现有 OS 上的脚本。
- 不追求"替代 Windows/macOS"。目标是新的计算场景：AI 原生的硬件交互、开发、部署。

---

## 判断成功的标准

不是因为有多少用户。不是因为融了多少钱。

**成功 = 一个 AI agent 能在 ShuoZi OS 上完成它在 Linux/Windows/macOS 上无法完成的事情。**

具体说：

1. AI 能自主发现、调用、组合系统中的任何 tool
2. AI 能在重启后恢复完整的工作上下文
3. AI 能直接与硬件交互（不需要人先装驱动）
4. 一个没写过的开发者，让 AI 给一块 ESP32 写并部署程序，全流程不超过 5 分钟
5. 系统 10 年持续积累，内核代码本身成为不可复制的竞争壁垒

---

## 不变的东西

技术会变，语言会变，硬件会变。但以下不变：

- **CLI 优先。** 永远不变。
- **能力 > 应用。** 系统暴露能力，不暴露软件。
- **agent 是用户。** 人类通过 agent 或 CLI 使用系统，不是通过 GUI。
- **安全是结构性的。** 不是事后加的权限检查，是架构本身的属性。
